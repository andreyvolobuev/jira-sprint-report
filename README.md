# Jira Sprint Reports

Инструменты для построения отчетов по спринтам в Jira.

## Настройка

### 1. Установка зависимостей

```bash
python -m venv venv
source venv/bin/activate
pip install requests
```

### 2. Настройка токена

Установите переменную окружения `JIRA_TOKEN`:

```bash
export JIRA_TOKEN='ваш_токен'
```

Или добавьте в `~/.bashrc` / `~/.zshrc` для постоянного использования.

## Использование

### Получить список спринтов

```bash
python list_sprints.py
```

Выводит:
- Активные спринты
- Последние 20 закрытых спринтов (с ID)
- Будущие спринты

### Получить отчет по спринту

```bash
python sprint_report.py -id <sprint_id>
```

Пример:
```bash
python sprint_report.py -id 6783
```

## Категории в отчете

| Категория | Описание |
|-----------|----------|
| **original** | Задачи в спринте на момент его активации (планировались) |
| **added_later** | Задачи добавленные после активации спринта (не планировались) |
| **carried_over** | Задачи переехавшие из предыдущих закрытых спринтов |
| **closed_planned** | Из original, закрыты до окончания спринта |
| **closed_unplanned** | Из added_later, закрыты до окончания спринта |

## Файлы

- `jira_client.py` — общие функции для работы с Jira API
- `list_sprints.py` — вывод списка спринтов
- `sprint_report.py` — генерация отчета по спринту

# Jira Sprint Reports

Скрипты для построения отчетов по спринтам в Jira.

## Настройка

1. Создать виртуальное окружение и установить зависимости:
```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

2. Установить переменную окружения с токеном:
```bash
export JIRA_TOKEN='ваш_токен'
```

## Использование

### 1. Получить список спринтов

```bash
python list_sprints.py
```

Выводит список всех спринтов доски: активные, закрытые (последние 20) и будущие.

Пример вывода:
```
АКТИВНЫЕ СПРИНТЫ
ID       Название                                 Период
----------------------------------------------------------------------
6784     26 Q1 S2 (26.01 - 08.02)                 2026-01-26 - 2026-02-09

ЗАКРЫТЫЕ СПРИНТЫ (последние 20)
ID       Название                                 Период
----------------------------------------------------------------------
6783     26 Q1 S1 (12.01 - 25.01)                 2026-01-12 - 2026-01-25
6295     Q4 S6 (15.12 - 28.12)                    2025-12-15 - 2025-12-31
...
```

### 2. Получить отчет по спринту

```bash
python sprint_report.py -id <sprint_id>
```

Пример:
```bash
python sprint_report.py -id 6783
```

Выводит отчет по указанному спринту с разбивкой по разработчикам.

**Категории задач:**
- **Планировались (original)** — задачи в спринте на момент его активации
- **Не планировались (added_later)** — задачи добавленные после активации спринта
- **Переехали из предыдущих спринтов (carried_over)** — задачи из предыдущих закрытых спринтов
- **Планировались и закрыты (closed_planned)** — из original, закрыты до окончания спринта
- **Не планировались, но закрыты (closed_unplanned)** — из added_later, закрыты до окончания спринта

В конце отчета выводятся агрегированные данные по всей команде.
